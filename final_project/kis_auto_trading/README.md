# KIS 자동매매 시스템

## 1. 프로젝트 개요

이 프로젝트는 한국투자증권 Open API를 이용하여 모의투자 환경에서 작동하는 자동매매 시스템을 구현한 것이다.

프로그램은 한국투자증권 API 서버에 접속하여 OAuth access token을 발급받고, 국내주식 현재가와 계좌 잔고를 조회한다. 이후 일봉 데이터를 바탕으로 이동평균선 전략을 적용하여 BUY, SELL, HOLD 신호를 생성하고, 조건이 충족되면 모의투자 주문을 실행한다.

이 repository는 인공지능과 금융공학 Final Project 제출을 위해 작성되었다.

## 2. 프로젝트 목표

이 프로젝트의 목표는 단순히 API 요청을 보내는 예제를 작성하는 것이 아니라, 실제로 작동하는 자동매매 흐름을 구현하는 것이다.

6. 위험관리 조건 확인
7. 모의투자 주문 실행

## 5. 환경변수 설정

실제 API key, API secret, 계좌번호는 `.env` 파일에 저장한다.

보안을 위해 `.env` 파일은 GitHub에 업로드하지 않는다. 대신 `.env.example` 파일에 필요한 형식만 기록한다.

예시:

```env
KIS_BASE_URL=https://openapivts.koreainvestment.com:29443
KIS_APP_KEY=YOUR_APP_KEY
KIS_APP_SECRET=YOUR_APP_SECRET
KIS_ACCOUNT_NO=YOUR_ACCOUNT_NUMBER
KIS_ACCOUNT_PRODUCT_CODE=01
```

각 변수의 의미는 다음과 같다.

| 변수                         | 의미                               |
| -------------------------- | -------------------------------- |
| `KIS_BASE_URL`             | 한국투자증권 모의투자 API 서버 주소            |
| `KIS_APP_KEY`              | KIS Developers에서 발급받은 App Key    |
| `KIS_APP_SECRET`           | KIS Developers에서 발급받은 App Secret |
| `KIS_ACCOUNT_NO`           | 모의투자 계좌번호 앞 8자리                  |
| `KIS_ACCOUNT_PRODUCT_CODE` | 계좌 상품 코드. 국내주식은 `01`을 사용한다.      |


이를 통해 한국투자증권 모의투자 API의 다음 기능이 Python 코드에서도 정상적으로 작동함을 확인하였다.

1. OAuth access token 발급
2. 국내주식 현재가 조회
3. 모의투자 계좌 잔고 조회
4. 모의계좌 예수금 및 보유 종목 확인

처음 Python 코드로 실행했을 때, 현재가 조회 직후 바로 잔고 조회를 실행하면서 다음 오류가 발생하였다.

```text
초당 거래건수를 초과하였습니다.
EGW00201
```

이 오류는 API 요청을 너무 빠르게 연속으로 보냈기 때문에 발생한 것으로 판단하였다. 따라서 현재가 조회와 잔고 조회 사이에 다음 코드를 추가하였다.

```python
time.sleep(1.5)
```

수정 후에는 잔고 조회가 정상적으로 실행되었다. 이 과정을 통해 한국투자증권 Open API에는 짧은 시간 안에 너무 많은 요청을 보내면 제한이 발생할 수 있으며, 자동매매 시스템에서는 API 호출 간격을 조절해야 한다는 점을 확인하였다.

또한 잔고 조회 과정에서 다음 오류도 확인하였다.

```text
ERROR INVALID INPUT_FIELD_SIZE
```

이 오류는 `.env` 파일의 `KIS_ACCOUNT_NO` 값에 계좌번호 전체 또는 잘못된 형식이 입력되었을 때 발생하였다. 한국투자증권 잔고 조회 API에서는 `CANO`에 계좌번호 앞 8자리만 입력해야 하므로, `.env`의 `KIS_ACCOUNT_NO` 값을 Postman에서 사용한 `CANO` 값과 동일하게 수정하였다.

수정 후 계좌 잔고 조회가 정상적으로 성공하였다.

따라서 1차 연결 테스트를 통해 다음 사항을 확인하였다.

| 항목                    | 결과                             |
| --------------------- | ------------------------------ |
| OAuth access token 발급 | 성공                             |
| 현재가 조회                | 성공                             |
| 잔고 조회                 | 성공                             |
| API 요청 제한 처리          | `time.sleep(1.5)`로 해결          |
| 계좌번호 입력 형식 오류 처리      | `KIS_ACCOUNT_NO`를 CANO 8자리로 수정 |
| 모의계좌 예수금              | 10,000,000원                    |
| 보유 종목                 | 없음                             |

이번 단계에서는 실제 주문을 실행하지 않고, 자동매매 시스템을 구성하기 위한 기본 API 연결이 정상적으로 작동하는지 확인하였다. 이후 단계에서는 일봉 데이터를 조회하고, 이동평균선 전략을 적용하여 BUY, SELL, HOLD 신호를 생성하도록 확장한다.

## 9. 매매 전략

이 프로젝트에서는 이동평균선 교차 전략을 사용한다.

사용하는 이동평균선은 다음과 같다.

* 단기 이동평균선: 5일 이동평균선
* 장기 이동평균선: 20일 이동평균선

매매 신호는 다음 기준으로 생성한다.

| 조건                              | 신호   |
| ------------------------------- | ---- |
| 5일 이동평균선이 20일 이동평균선을 아래에서 위로 돌파 | BUY  |
| 5일 이동평균선이 20일 이동평균선을 위에서 아래로 돌파 | SELL |
| 명확한 교차 신호가 없음                   | HOLD |

이 전략을 선택한 이유는 구조가 단순하고, 현재가 조회·일봉 조회·전략 판단·주문 실행까지 자동매매 시스템의 전체 흐름을 설명하기에 적합하기 때문이다.

### 9.1 이동평균선 전략 Python 실행 결과

일봉 데이터 조회 API를 Python 코드로 구현한 뒤, 조회한 종가 데이터를 이용하여 5일 이동평균선과 20일 이동평균선을 계산하였다.

실행 명령어는 다음과 같다.

```bash
python main.py
```

실행 결과는 다음과 같다.

```text
[4] 일봉 데이터 조회 중...
일봉 데이터 조회 성공

[5] 이동평균선 전략 분석 중...

[이동평균선 전략 분석 결과]
기준일자: 20260619
종가: 354,000원
5일 이동평균선: 348,600.0원
20일 이동평균선: 324,450.0원
매매 신호: HOLD
판단 이유: 이동평균선 교차 신호가 발생하지 않았습니다.

2차 전략 테스트 완료
```

이번 실행에서는 한국투자증권 Open API를 통해 일봉 데이터를 정상적으로 불러왔고, 해당 데이터에서 날짜와 종가를 추출하였다. 이후 `strategy.py`에서 종가 데이터를 날짜 기준 오름차순으로 정렬한 뒤, `pandas`의 rolling mean 기능을 사용하여 5일 이동평균선과 20일 이동평균선을 계산하였다.

전략 판단 결과, 기준일자 `20260619`의 종가는 `354,000원`이었다. 5일 이동평균선은 `348,600.0원`, 20일 이동평균선은 `324,450.0원`으로 계산되었다.

현재 5일 이동평균선이 20일 이동평균선보다 위에 있지만, 직전 거래일과 현재 거래일 사이에서 새롭게 교차가 발생한 것은 아니므로 `BUY` 신호가 아니라 `HOLD` 신호가 생성되었다.

즉, 이번 상황에서는 이미 단기 이동평균선이 장기 이동평균선 위에 있는 상태였지만, 새로 매수할 만한 교차 신호는 발생하지 않았다고 판단하였다. 따라서 자동매매 시스템은 불필요한 매수 또는 매도 주문을 실행하지 않고 관망하는 것이 적절하다고 판단하였다.

이번 단계에서 확인한 내용은 다음과 같다.

| 항목                      | 결과             |
| ----------------------- | -------------- |
| 일봉 데이터 조회               | 성공             |
| 종가 데이터 추출               | 성공             |
| 날짜 기준 정렬                | 성공             |
| 5일 이동평균선 계산             | 성공             |
| 20일 이동평균선 계산            | 성공             |
| BUY / SELL / HOLD 신호 생성 | 성공             |
| 현재 매매 신호                | HOLD           |
| 판단 이유                   | 이동평균선 교차 신호 없음 |

이번 2차 테스트를 통해 단순히 API 연결만 되는 것이 아니라, API로 받은 시장 데이터를 실제 전략 판단에 사용할 수 있음을 확인하였다. 이후 단계에서는 이 전략 신호를 `risk_manager.py`와 연결하여, 주문을 실행해도 되는 상황인지 검사한 뒤 모의투자 주문으로 이어지도록 확장한다.

## 10. 위험관리

자동매매 시스템은 주문을 실행하기 전에 기본적인 위험관리 조건을 확인한다.

위험관리 규칙은 다음과 같다.

1. 실전투자가 아닌 모의투자 환경만 사용한다.
2. 한 번의 주문에 사용할 수 있는 최대 금액을 제한한다.
3. 현금 잔고가 부족하면 매수 주문을 실행하지 않는다.
4. 이미 보유 중인 종목에 대해 중복 매수를 방지한다.
5. 보유 수량이 없으면 매도 주문을 실행하지 않는다.
6. 모든 매매 판단과 주문 결과를 log로 저장한다.

이러한 조건을 통해 의도하지 않은 주문이나 과도한 주문을 방지한다.

## 13. Oral Test 대비 코드 설명

### 13.1 전체 작동 흐름

프로그램은 `main.py`에서 시작한다.

`main.py`는 환경변수를 불러오고, `kis_api.py`를 통해 access token을 발급받는다. 이후 현재가, 잔고, 일봉 데이터를 조회하고, `strategy.py`에서 매매 신호를 생성한다. 생성된 신호는 `risk_manager.py`에서 주문 가능 여부를 검사한 뒤, 조건이 충족되면 모의투자 주문을 실행한다. 마지막으로 `logger.py`를 이용해 실행 결과와 거래 기록을 저장한다.

### 13.2 `kis_api.py`

`kis_api.py`는 한국투자증권 Open API와 직접 통신하는 파일이다.

주요 함수는 다음과 같다.

* `get_access_token()`: access token을 발급받는다.
* `get_current_price()`: 특정 종목의 현재가를 조회한다.
* `get_balance()`: 모의투자 계좌 잔고를 조회한다.
* `get_daily_prices()`: 일봉 데이터를 조회한다.
* `place_order()`: 모의투자 주문을 실행한다.

API 관련 기능을 별도 파일로 분리하여, 전략 코드와 API 통신 코드를 구분하였다.

### 13.3 `strategy.py`

`strategy.py`는 매매 전략을 담당한다.

일봉 종가 데이터를 바탕으로 5일 이동평균선과 20일 이동평균선을 계산하고, 두 이동평균선의 교차 여부에 따라 BUY, SELL, HOLD 신호를 생성한다.

### 13.4 `risk_manager.py`

`risk_manager.py`는 전략이 생성한 신호를 실제 주문으로 연결해도 되는지 확인한다.

예를 들어 BUY 신호가 발생하더라도 현금이 부족하거나 이미 종목을 보유하고 있으면 주문을 제한할 수 있다.

### 13.5 `logger.py`

`logger.py`는 자동매매 시스템의 실행 결과를 저장한다.

거래 기록은 Final Project 요구사항 중 하나이므로, 주문 결과와 매매 판단 이유를 log로 남기는 것이 중요하다.
